# 鲁棒性测试工具使用指南

## 概述

本工具用于测试DCT水印算法抵抗各种图像攻击的能力。通过模拟常见的图像处理操作，验证水印在不同攻击下的可提取性。

---

## 快速开始

### 1. 准备测试环境

确保已编译DCT水印工具：

```bash
gcc -DDEBUG_MODE -O2 -o dct_watermark_tool.exe dct_watermark.c -lm
```

### 2. 创建测试图像

```bash
# 使用任意24位BMP图像，或创建测试图像
# 示例：512x512的彩色渐变图像
```

### 3. 嵌入水印

```bash
dct_watermark_tool.exe encode test_image.bmp watermarked.bmp "Your Watermark Text" 30.0
```

### 4. 运行鲁棒性测试

```bash
# 编译测试工具
gcc -O2 -fexec-charset=UTF-8 -o robustness_test_v2.exe robustness_test_v2.c -lm

# 运行测试
robustness_test_v2.exe watermarked.bmp
```

## 测试类型说明

### 1. 裁剪攻击 (Crop Attack)
模拟图像被部分裁剪的情况。

- **保留75%**: 裁剪掉25%的边缘
- **保留50%**: 裁剪掉50%的边缘
- **保留25%**: 裁剪掉75%的边缘（仅保留中心区域）

**预期**: 由于水印采用全图冗余嵌入，应该能够抵抗裁剪攻击。

---

### 2. 噪声攻击 (Noise Attack)
向图像添加随机噪声，模拟传输过程中的干扰。

- **级别10**: 轻微噪声 [-10, +10]
- **级别20**: 中等噪声 [-20, +20]
- **级别30**: 较强噪声 [-30, +30]
- **级别50**: 强烈噪声 [-50, +50]

**预期**: 低级别噪声不应影响水印提取，高级别噪声可能会破坏水印。

---

### 3. 模糊攻击 (Blur Attack)
对图像进行模糊处理，测试水印对平滑操作的抵抗力。

- **核大小3**: 轻微模糊 (3x3均值滤波)
- **核大小5**: 中等模糊 (5x5均值滤波)
- **核大小7**: 强烈模糊 (7x7均值滤波)

**预期**: 模糊会改变像素的局部关系，可能会影响水印提取。

---

### 4. 亮度调整 (Brightness Adjustment)
调整图像整体亮度，测试对光度变化的鲁棒性。

- **系数0.5**: 降低亮度至50%
- **系数0.7**: 降低亮度至70%
- **系数1.5**: 提高亮度至150%

**预期**: 适度亮度变化不应影响水印，极端变化可能会破坏水印。

---

### 5. 缩放攻击 (Resize Attack)
改变图像尺寸，测试对缩放的鲁棒性。

- **50%**: 缩小至一半
- **75%**: 缩小至3/4
- **150%**: 放大至1.5倍

**预期**: 缩放操作应该不影响水印提取，因为DCT域的相对关系会保留。

---

### 6. JPEG压缩攻击 (JPEG Compression)
使用不同质量进行JPEG压缩，测试对有损压缩的抵抗力。

- **质量95**: 高质量压缩
- **质量90**: 标准高质量
- **质量80**: 中等质量
- **质量70**: 较低质量

**预期**: 由于JPEG也使用DCT，可能会有干扰。高质量应该能保持水印。

## 输出结果说明

### 测试报告格式

```
测试项                 状态   相似度  解码内容
------------------------- -------- ---------- --------------------
基准测试-原始图像        通过   100%    Robustness Test Mess...
裁剪攻击-75%            通过   85%     Robustness Test Mess...
噪声攻击-级别30         失败   0%      [无法解码]
```

### 字段说明

- **测试项**: 攻击类型和参数
- **状态**: 通过/失败（是否能检测到水印）
- **相似度**: 解码内容与原始水印的相似度（字符级别）
- **解码内容**: 实际提取的水印文本（前30个字符）

## 评估标准

### 优秀 (通过率 >= 90%)
水印具有极强的鲁棒性，能够抵抗绝大多数攻击。

### 良好 (通过率 70%-89%)
水印具有良好的鲁棒性，适合一般应用。

### 中等 (通过率 50%-69%)
水印具有基本的鲁棒性，建议提高嵌入强度。

### 较差 (通过率 < 50%)
水印鲁棒性不足，需要调整算法参数或增加纠错机制。

## 常见问题

### Q1: 为什么基准测试失败？
**A**: 确保传入的路径是正确的含水印图像路径。测试程序会先复制图像到测试目录。

### Q2: 如何提高水印鲁棒性？
**A**: 
1. 增加嵌入强度（`strength`参数）
2. 使用更稳定的DCT系数位置
3. 添加纠错编码（如Reed-Solomon码）
4. 增加冗余度（多系数嵌入）

### Q3: 某些测试通过，某些失败，正常吗？
**A**: 完全正常。不同攻击对水印的影响不同。关键是根据应用场景选择需要抵抗的攻击类型。

### Q4: 相似度很低但测试通过，说明什么？
**A**: 说明水印信号仍然存在，但提取的内容有误。这可能是因为攻击改变了某些位，但同步标记仍能被检测到。

## 高级用法

### 自定义测试参数

编辑 `robustness_test_v2.c` 中的 `test_defs` 数组：

```c
struct {
    char name[64];
    int type; // 0=crop, 1=noise, 2=blur, 3=brightness, 4=resize, 5=jpeg
    float param;
} test_defs[] = {
    {"自定义测试", 1, 40},  // 噪声级别40
    // ...
};
```

### 添加新测试类型

1. 添加攻击函数实现
2. 在 `test_defs` 中添加新类型
3. 在 `main()` 函数的 `switch` 中添加新分支
